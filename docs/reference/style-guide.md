## Style Guide

This is a minimal style guide for consistent fake documentation.

## Headings

- Use `##` for the page title
- Use `###` for major sections

## Tone

- Short, instructional sentences
- Prefer checklists and examples

## Formatting

- Use code blocks for commands and snippets
- Use tables sparingly and only when it improves scanning

## Example snippet

```text
Keep examples small and copy/pastable.
```
