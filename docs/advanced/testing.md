## Testing Strategy

This is a **fake testing guide** for documentation tooling.

## What to test

- **Links**: every TOC link points to an existing file
- **Headings**: your renderer indexes headings correctly
- **Navigation**: relative links work across directories
- **Search**: common terms appear where expected

## Example “smoke test” checklist

- [ ] Click through every TOC entry
- [ ] Follow at least one cross-link per page
- [ ] Confirm code blocks render with monospaced fonts
